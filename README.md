# 学院专题学习助手

本目录中的 `gdgbpx-workshop-helper.user.js` 是针对新版“专题学习”Vue 页面重新编写的独立 Tampermonkey 脚本，不依赖三份旧脚本。

## 使用方式

1. 直接打开 [GitHub 安装链接](https://raw.githubusercontent.com/Linkegee/gdgbpx-workshop-helper/main/gdgbpx-workshop-helper.user.js)，由 Tampermonkey 安装；不要复制代码后手动新建脚本。
2. 停用针对同一网站的旧脚本，避免多个脚本同时点击或刷新。
3. 用户自行登录并完成课程报名。
4. 手动进入“专题学习 → 在学”。
5. 在左下角悬浮面板点击“开始”；可拖动标题栏移动面板。脚本固定使用正常 `1×` 速度以保证服务器记满学习时长。

脚本支持开始、暂停、继续、停止、跳过当前课程、重新检查进度、静音、自动恢复普通暂停和分页处理。控制面板默认贴在左下角，可拖动标题栏移动，并会记住位置。

## 自动更新

- 脚本的 `@updateURL` 和 `@downloadURL` 均指向 GitHub `main` 分支，每次发布必须提升 `@version`。
- Tampermonkey 5.5 及以上版本需要在扩展设置中同时开启定期更新检查和“自动安装（Automatic installation）”；只开启检查不会自动安装。
- 脚本自身每天还会检查一次 GitHub 版本；发现新版时，助手面板会显示“安装更新”按钮。Tampermonkey 菜单中也可手动执行“检查脚本更新”。
- 浏览器安全策略不允许用户脚本静默改写自身；如果 Tampermonkey 的自动安装关闭，用户仍需点击一次面板中的安装按钮。

## 诊断日志

脚本会把主站、播放器外层页面和播放器 iframe 的日志汇总保存，最多保留最近 600 条。日志包括：

- 脚本启动、页面地址和运行状态变化；
- 专题与课程列表的读取结果、点击和分页；
- 视频源、`readyState`、`networkState`、时长和播放进度；
- 元数据加载、开始播放、暂停、缓冲、卡死、播放失败和视频错误；
- 播放器窗口关闭、专题刷新以及未捕获的 JavaScript 异常。

面板底部提供三个操作：

- **复制**：把当前状态和全部日志复制到剪贴板；
- **下载**：生成一个 `gdgbpx-helper-log-日期.json` 文件；
- **清空**：删除旧日志，方便重新复现问题。

出现问题时，先不要刷新或清空日志，点击“下载”，然后把生成的 JSON 文件发来即可。脚本会自动遮盖常见的 Token、Cookie、授权字段、会话参数和用户标识；分享前仍建议自行快速检查一遍文件内容。

Tampermonkey 菜单中也提供“复制诊断日志 / 下载诊断日志 / 清空诊断日志”，面板被隐藏时仍可使用。

### 自动保存到本机供 Codex 检查

双击运行项目中的 `start-debug-bridge.cmd` 后，脚本会把已经脱敏的日志自动发送到：

`logs/gdgbpx-debug.ndjson`

调试桥只监听 `127.0.0.1:17891`，不接受局域网或互联网连接，并要求 Tampermonkey 请求携带专用请求头。连接失败不会影响课程播放，日志仍会暂存在 Tampermonkey 中；恢复连接后会补发最近等待发送的记录。

面板显示“本机调试桥：已连接，日志自动保存”后，出现问题只需告诉 Codex大致发生时间和现象，Codex 可以直接读取项目日志，不需要再复制或下载文件。

## 隔离 Chrome 远程调试

双击 `start-debug-chrome.cmd` 会使用 `%LOCALAPPDATA%\CodexChromeProfiles\gdgbpx-debug` 作为独立用户目录启动 Chrome，并只在本机 `127.0.0.1:9222` 开放 Chrome DevTools Protocol。这个 Chrome 不复用日常 Chrome 的 Cookie、扩展、历史记录或登录状态；首次使用需要单独安装 Tampermonkey、安装本脚本并登录学院。

运行 `powershell -ExecutionPolicy Bypass -File .\chrome-debug.ps1 -Action status` 可以检查连接；运行 `node .\chrome-cdp-snapshot.js` 会把学院主页面和播放器页面的 DOM、视频、弹窗、控制台及加载失败快照保存到 `logs\chrome-cdp-snapshot-latest.json`。快照不读取或保存 Cookie 内容。关闭这个专用 Chrome 窗口即可结束调试。

### v1.2.1 修复

播放器事件现在绑定到打开它的具体课程。上一节播放器窗口延迟发出的“已关闭”等事件，如果此时已经进入下一课，会被识别为过期事件并忽略，避免下一课刚启动就被旧事件打断。

### v1.2.2 修复

无播放进度默认只告警并继续尝试恢复，不再自动关闭播放器。面板新增“卡死后自动重开”开关，只有用户明确开启后，达到设定的无进度时间才会关闭并重新打开课程。这可避免站点短暂暂停或异常缓冲时被误判为必须关闭。

### v1.2.3 修复

视频原生 `ended` 或网站结束提示出现后，脚本等待 4 秒供站点保存完成记录，然后明确关闭播放器窗口，不再完全依赖网站的自动关闭行为。父页面随后刷新进度并进入下一课。

### v1.2.4 修复

脚本只把 `classType=3` 识别为“在学”列表，不再把“进行中”等其他专题列表的临时空内容误判为全部完成。“在学”列表没有课程卡片时还会等待至少 8 秒并确认加载状态，避免 Vue 页面切换期间的空 DOM 导致播放器收到错误的 `complete` 状态而暂停。

### v1.3.0 计时策略修复

实测确认服务器按实际学习时长而非单纯的视频播放位置累计课程进度，因此脚本固定使用 `1×`。如果旧版倍速已经把播放器记忆位置写到结尾、但服务器仍显示未完成，新版会把视频回退到服务器已确认进度附近并保留 30 秒重叠，以正常速度补足剩余学习时间。所有主动关闭播放器的路径现在都会记录明确的关闭原因。

### v1.3.1 防循环修复

播放器即便在加载完成后才被网站跳到结尾，原生 `ended` 事件也会再次检查服务器已确认进度；未完成时立即回退继续，不会关闭。播放器连续三次未能启动时，脚本会暂停并提示检查浏览器弹窗权限，而不是无限点击课程入口。

### v1.5.11 后台节流与旧会话尾随事件修复

播放器进入后台后，Chrome 可能把播放器页的 1 秒定时器节流到接近 60 秒。关闭请求现在由共享状态变更监听器立即处理，不再等待播放器定时器；主页面也不再仅凭心跳超时判断窗口已关闭，必须先收到对应播放器的页面离开信号，再确认该会话心跳停止。已经由服务器确认完成的旧播放器即使迟到发送播放、进度或关闭事件，也不会再覆盖“准备下一节”的状态。升级时若检测到旧版本因此留下的“暂停且当前课程为空”状态，会自动恢复并继续选择下一节。

### v1.5.12 播放器启动导航与过期复查定时器修复

播放器外层页从“验证中”跳转到正式播放页时会正常触发一次 `beforeunload`。脚本现在把这次零时长、零进度的验证页离开识别为启动导航，不再误判为播放器关闭。视频开始或继续产生进度后会主动取消先前遗留的完成复查定时器并清零复查次数；复查回调执行前还会再次核对当前阶段，已经恢复播放时只记录并忽略，避免面板在“播放中”和“等待已完成”之间反复跳变。

### v1.5.14 课程专属播放器兜底与身份确认

常规弹窗被浏览器拦截时，扩展标签页兜底不再打开会复用上一节上下文的固定播放器地址，而是从专题页面读取站点自己的 `playDomain`、当前课程 `courseId` 和登录授权，构造与人工点击完全一致的课程专属验证地址。播放器在页面可见标题与当前待学课程核对一致前，不会上报“已开始”或进度；若检测到旧播放器被误复用，会立即记录身份不匹配并关闭旧窗口，避免主页面把上一节误认成下一节。

### v1.5.15 修正播放器资源标识映射

网站课程卡片同时包含数据库 `courseId` 和播放器 `resourceCode`，实际点击处理为 `jump(resourceCode, courseId)`，验证页虽把参数命名为 `courseId`，实际要求传入 `resourceCode`。兜底播放器现严格复刻这一调用约定，避免下一节页面出现“没有找到播放资源”而无法继续。

### v1.5.16 下一节播放器改用可靠主路径

实测网站课程标题的原生点击会偶发连续不创建播放器窗口，旧版要经过三轮各 30 秒的等待后才启用扩展标签页兜底，造成上一节完成后长时间看似没有继续。现在脚本在选定下一节并写入课程身份后，立即按网站 Vue 数据中的 `playDomain`、授权 token 与 `resourceCode` 使用 Tampermonkey 管理的标签页打开播放器；网站原生点击仅在无法取得课程播放器上下文时备用。播放器身份、服务器“已完成”判定和关闭确认规则均保持不变。

### v1.5.13 后台暂停即时恢复

Chrome 把播放器窗口置于后台后，页面的 1 秒轮询定时器可能被节流，导致网站短暂停止视频时无法及时触发原有自动恢复。播放器现在直接在原生 `pause` 事件中检查运行状态并立即调用 `play()`，不再等待轮询定时器。用户通过助手暂停、停止、跳过、服务器已确认完成或页面出现课程提问时均不会自动恢复。

### v1.5.9 可见详情局部同步

播放期间仍不刷新整个 Vue 页面。后台同源探针每次取得服务器最新课程列表后，会按课程标题把可见详情页中的百分比文字、进度条、“已完成/未完成”状态以及“必修（已完成数/总数）”局部同步；这不会重置脚本、定时器、播放器会话或当前课程上下文。课程结束判定仍只采用服务器返回的明确“已完成”状态，百分比只负责展示。

### v1.5.8 播放器会话身份与关闭心跳确认

播放器会在自己的会话存储中固定课程身份，并从播放器页面可见标题反查真实课程，旧窗口不会因为主面板切换而被错误标成下一课。`beforeunload` 只表示页面正在离开，不再直接视为播放器已经关闭；只有对应课程与会话的播放器心跳连续 10 秒消失后，主页面才允许选择下一课。

### v1.5.7 探针进度显示优先级

播放期间，后台探针读取到的服务器进度优先于用户当前可见详情页的旧 DOM；可见页面中的较低百分比不再把面板进度覆盖回 `0%`。明确的“已完成”状态仍然可以立即触发完成确认。

### v1.5.6 自动更新检查与选课决策日志

1.5.5 额外记录每次选课扫描的 `lessonKey`、网页状态、百分比、排除原因（网页已完成/服务器完成记忆/手动跳过）和最终点击候选，并把完成关闭请求与对应课程 key 关联，便于区分“服务器未完成”“播放器未关闭”和“选错旧 DOM”三类问题。

播放过程中不再刷新用户当前看到的专题详情页。脚本会在同源后台 iframe 中每约 6.5 秒重新读取一次专题详情，并在 Vue DOM 尚未渲染完成时继续重试读取；只把当前课程的明确状态文本 `已完成` 作为关闭播放器和进入下一节的条件；百分比仅用于面板显示和诊断。服务器确认完成的课程会被记录在本轮状态中，即使可见详情页仍显示旧百分比，也不会再次选择同一课程。如果视频结束或播放器提前关闭但服务器仍未标记 `已完成`，脚本会留在当前课程并继续复查，不会直接跳到下一节。

## 播放与进度原则

- 固定使用 `1×`。实测站点按实际学习时长累计进度，`1.5× / 2×` 会出现视频已结束但课程进度未满的情况。
- 不向前拖动进度、不调用或伪造完成接口、不修改课程标识和心跳数据；仅在旧版倍速留下“播放器已在结尾、服务器仍未完成”的异常状态下，回退到服务器已确认位置附近继续正常播放。
- 正常等待视频 `ended`；“卡死重试”仅在播放器持续数分钟没有任何时间进度时关闭播放器并重新检查，不是固定播放三分钟就关闭。
- 如果课程出现内嵌提问，脚本暂停自动恢复并提示用户手动作答。
- 自动播放会等待站点装载有效视频源和元数据，再点击 Video.js 的播放按钮；不会在空播放器阶段反复调用 `play()`。

## 已确认的网站行为

- 播放器每 30 秒更新一次 `lesson_location` 和实际播放 `session_time`。
- 外层页面每 60 秒向服务器发送一次心跳。
- 视频触发 `ended` 后，站点调用自身完成接口；可通过后台同源详情探测页读取最新的 `已完成` 状态，不打断正在播放的窗口。

## 首次实测重点

- 浏览器是否拦截脚本点击后产生的播放器窗口。
- 视频结束后，日志中是否出现 `server-status-probe-result`，并且只有读取到 `status=已完成` 后才关闭播放器、进入下一课。
- “暂停/继续/停止/跳过当前”能否跨播放器窗口生效。
- 课程全部完成后，专题是否从“在学”移到“已学”，脚本是否继续下一专题。
